## Plan — Article Workflow Security & Reliability Hardening

**Status: ✅ Completed — 2026-05-28**

### Problems Identified

1. **Hardcoded webhook secret**: `lib/agentHelper.ts` had `?secret=sherly` hardcoded in source, visible in git history to anyone with repo access. Impossible to rotate without a code deploy.
2. **No TTL on draft keys**: `article:{topic}` Redis keys had no expiry. Orphaned `generating` or `error` drafts accumulated forever, cluttering the admin dashboard and consuming Upstash storage quota.

### Changes Made

#### `lib/agentHelper.ts`
- Removed hardcoded `CALLBACK_URL` constant.
- Added `buildCallbackUrl()` function that reads from env vars in priority order:
  1. `DASHBOARD_CALLBACK_URL` (full override)
  2. `WEBHOOK_SECRET` or `ADMIN_SECRET` + `NEXT_PUBLIC_SITE_URL`
  3. Throws a clear `Configuration Error` if no secret is found (fail-closed).
- `generating` drafts now set with `ttlSeconds: 604800` (7 days).
- `error` drafts now set with `ttlSeconds: 172800` (2 days).

#### `lib/draftsDb.ts`
- Added `SetDraftOptions` interface with `ttlSeconds?: number`.
- `setDraft()` now accepts optional `options` parameter.
- When `ttlSeconds` is provided, uses Upstash REST `/setex/{key}/{ttl}` instead of plain `/set`.
- `pending_review` and `published` drafts still use plain `/set` (no TTL — must survive until admin action).

#### `.env.local` & `.dev.vars`
- Added `WEBHOOK_SECRET=sherly` (same value as `ADMIN_SECRET` for backwards compatibility).
- Added `NEXT_PUBLIC_SITE_URL=https://arabiakhaleej.com`.

### Required Cloudflare Pages Env Var Updates
Add in **Cloudflare Pages → Settings → Environment Variables**:
- `WEBHOOK_SECRET` — same value as your `ADMIN_SECRET`
- `NEXT_PUBLIC_SITE_URL` — `https://arabiakhaleej.com`
